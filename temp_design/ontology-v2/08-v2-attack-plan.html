Aryx Enterprise · v2 · the attack plan

Build order — foundation first, converter second, the rest in waves.

Six phases, prioritized so (1) the Oracle move later is a config swap, not surgery, (2) the screen that converts the skeptic ships early, and (3) nothing visible is faked. Branch: feat/ontology-v2-enterprise.

0The one rule that orders everything

🧬

Build the seam before the features — or Aryx-o becomes spinal surgery.

Every v2 feature must call a capability port, never a concrete store. If the Map talks to FalkorDB directly, then Aryx-o = rewrite the Map. If it talks to a GraphPort, then Aryx-o = write an OracleRdfAdapter + flip config. Phase 0 wraps today's Postgres/FalkorDB/Ollama as the default adapters first, so all of v2 is built on the seam and v2.1 is a swap.

2Tonight's target — honest scope

What's actually buildable tonight

Phase 0 (the seam) + Phase 1 (the Accuracy Lab).

  1. Phase 0 — the ports & adapters seam. Define the 6 ports, wrap the current stores as default adapters, config-driven loading. This is the foundation; it makes every later phase clean and Aryx-o cheap. Invisible but decisive.
  2. Phase 1 — the Accuracy Lab. The groundedness + reasoner-check engine, the fair A/B, and the /lab screen. The thing that converts Magnus and forces the provenance engine into being.
Honest framing (Red's rule): v2 Enterprise in full is weeks, not a night — it's the whole design set. Tonight we lay the seam and ship the one screen that proves the thesis. Phases 2–5 follow in waves, each shipping only when its checklist rows (07) go green. No faked timelines, no faked screens.
