Cost Governor — live, automatic.

Per-tier token budgets throttle LLM concurrency. At 80% of the frontier budget, the Governor auto-downgrades frontier→cheap so a job never blows the bill. 429 rate-limits trigger exponential backoff. Per-doc 300s + per-call 120s timeouts prevent any single item wedging the run.

🔢 Bounded concurrency

Workers autoscale to a CAP (e.g. 32). LLM calls throttled by the token Governor + per-source rate limits. Queue depth is the back-pressure signal — never unbounded fan-out.

satisfies · Red (no runaway)

💲 Cost, live + projected

Every phase shows calls · tokens · $ spent · % of budget. Projected monthly on the header meter. The Governor's downgrade is visible, not silent.

satisfies · Eleanor

👁 Per-phase observability

Tasks / in-flight / done / failed / throughput / ETA per phase, plus per-prediction counters (tag · embed · extract · adjudicate · deliberate). Stuck-stage + budget-breach alerts.

satisfies · Sven · Magnus
The honest answer to "how many parallel runs?": as many as the worker cap and the cost Governor allow — and not one more. At 200M rows the system isn't running 200M predictions; it's running bounded, sampled, governed task waves, each metered to the dollar, each watchable per phase, each recoverable. That's the difference between a pipeline and a money fire.
