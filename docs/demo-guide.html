Hook up Claude Desktop

One config block, 21 tools, restart

Use the local Claude Desktop app — claude.ai (the web) blocks raw-HTTP MCP connectors. Pre-req: open 8765/tcp inbound on the EC2 security group, same as 3000 and 8088.

# 1. Verify MCP is reachable from your laptop:
curl -m 6 -i http://ec2-3-91-73-197.compute-1.amazonaws.com:8765/sse
# expect text/event-stream — if it times out, open SG port 8765.

# 2. Edit (or create) on macOS:
# ~/Library/Application Support/Claude/claude_desktop_config.json

{
  "mcpServers": {
    "aryx": {
      "command": "npx",
      "args": [
        "-y", "mcp-remote",
        "http://ec2-3-91-73-197.compute-1.amazonaws.com:8765/sse"
      ]
    }
  }
}

# 3. Quit Claude Desktop fully (⌘Q), reopen. New chat → 🔌 icon → 21 tools.

# ────────────────────────────────────────────────────────────────────
# Want claude.ai web (not Desktop)? You need HTTPS.
# Quick option — Cloudflare tunnel on EC2:
cloudflared tunnel --url http://localhost:8765
# Use the printed https://xxx.trycloudflare.com/sse URL in
# claude.ai → Settings → Connectors → Custom MCP.
