🏗️ Aryx — Technical Flow Architecture

4-Layer system design with components and data flow

Architecture Overview

  • UI Layer: Streamlit (web interface) and FastAPI (REST API) accept user requests
  • Orchestration Layer: Pipeline runs 7-stage ingest; LLM Broker routes queries; Job Queue manages async work; Workspace Manager handles multi-tenancy
  • Models Layer: Local Ollama handles cheap tagging & scoring; frontier APIs (Claude, OpenAI) handle hard decisions (~1–5%); embeddings for semantic search
  • Storage Layer: PostgreSQL = source of truth (all records, provenance, workspace isolation); FalkorDB = interactive graph (rebuilt from Postgres as needed); Redis = session cache
UI: User interface
Orchestration: Task coordination
Models: LLM providers
Storage: Data persistence
