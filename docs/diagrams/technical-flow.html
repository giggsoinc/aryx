Aryx — Technical Flow Architecture

5-layer system: UI, orchestration, HITL, models, storage

Architecture Overview

  • UI Layer: Streamlit web UI, FastAPI REST (23 routers), MCP server (3 tools over SSE), adjudication + actions UIs
  • Orchestration: 7-stage pipeline with StageTracker checkpoints; resolution funnel (block → score → 4-band route → cluster → golden); action engine; incremental projection; workspace isolation with survivorship policies
  • HITL Layer: Adjudication queue (review-band pairs), ontology approval gate, action approval (MCP always-pending), survivorship configuration
  • Models: Ollama (local, free) for cheap stages; Claude/OpenAI/Gemini for frontier decisions (~1-5%); rules engine for guard conditions
  • Storage: PostgreSQL = source of truth (23 idempotent migrations, LIST-partitioned, psycopg3 pool); FalkorDB = rebuildable graph projection; all SQL in .sql files (DB-Guard enforced)
