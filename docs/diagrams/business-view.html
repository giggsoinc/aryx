Aryx — Business View Architecture

User paths, HITL gates, and the knowledge graph

How It Works

  • Ingest path: Connect database or upload files → auto-discover entity types → resolve duplicates via four-band funnel → build graph
  • Ask path: Natural-language question → LLM retrieves entities, reasons, returns answer with source provenance
  • Graph path: Explore entities visually → drill down to relationships, confidence scores, provenance chain
  • MCP path: AI agents browse (list), query (ask), or request mutations (act) — actions always require human approval
  • HITL loop: Ambiguous pairs queue for human steward; decisions persist as labeled training data (data moat)
  • Actions: Declarative mutations with guard conditions, human approval gate, and full audit trail
