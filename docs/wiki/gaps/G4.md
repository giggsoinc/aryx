## Problem

`_bearer_ok` in `main.py` returned `True` on exception (fail-open), and there was no API-key enforcement layer for the REST surface. Any network error to the token store would silently admit requests.

## Root Cause (file:line)

- `src/aryx/api/main.py:49` — `except Exception: return True` (pre-fix)
- No `ApiKeyMiddleware` existed; `ARYX_API_AUTH` env var was ignored

## Fix

1. **`src/aryx/api/security.py`** (new) — `ApiKeyMiddleware(BaseHTTPMiddleware)`:
   - Reads `X-Aryx-Api-Key` header
   - `ARYX_API_AUTH=off|optional|required` (default: `optional`)
   - Exempt paths: `/health`, `/docs`, `/openapi.json`, `/redoc`, `/mcp*`
   - Required mode + bad/missing key → 401 + `WWW-Authenticate: ApiKey`
   - Optional mode + invalid key → `X-Aryx-Auth-Warning` header, passes through
   - `_verify_key` fails closed on any exception (returns `False`)

2. **`src/aryx/api/main.py`** — exception handler changed to `return False`; `create_app()` adds `ApiKeyMiddleware`; `_lifespan` wired
