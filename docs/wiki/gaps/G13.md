## What Was Built

The smallest honest kinetic ontology: declared, parameterized, permissioned
mutations, every execution audited, **Postgres first, graph second** —
FalkorDB is never mutated directly (core architecture rule, DEC-004).

- **DSL** — JSON; `guard` reuses the rules-engine `when` matcher VERBATIM
  (`actions/engine.py` imports `reasoning.engine._match`). One condition
  grammar across rules, axioms, actions. Effects v1: `set_attribute`,
  `add_relationship`, `remove_relationship`, `set_label`. No entity deletes,
  no source-system writeback (v2).
- **Migration 0023** — `aryx_action` (versioned by append, `superseded_by`
  pointer per the 0014 change-log precedent) + `aryx_action_execution`
  (`effect_log` JSONB records before/after per effect — kinetic provenance).
- **`store/action_store.py`** — definitions, executions, and the
  Postgres-first effect targets (attribute JSONB merge touches `updated_at`
  → G8 dirty set; relationship rows insert/delete).
- **API** — CRUD, `POST /actions/{name}/execute` (guard → params →
  pending|auto-apply), `GET /actions/executions`,
  `POST /actions/executions/{id}/decide` — mirrors G10's decide shape
  verbatim; a steward learns one workflow. Failures are audited
  (`status=failed` + error in effect_log), never silent.
- **MCP `act` tool (DEC-005)** — external agents may REQUEST an action;
  the execution is ALWAYS pending regardless of the action's approval flag.

## Demo (the Palantir-tier conversation opener)

`flag_sole_source_supplier`: guard `alternate_count == 0` → effects set
`risk_flag=SOLE_SOURCE` + `ESCALATED_TO → Procurement`; steward approves
via decide; graph shows the edge after re-projection; the execution row is
the full audit. The exact definition is the fixture in tests/test_actions.py.
