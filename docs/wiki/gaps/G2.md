# G2 - Naive Blocking
**Agent:** block-smith  **Status:** DONE  **Branch:** gap/g2-block

## Verified Baseline

The original `block()` in `src/aryx/resolution/classical.py` used a single
prefix key (first 4 normalised characters).  This caused two classes of missed
matches:

* **Transpositions** - "John Smith" and "Smith John" produce different prefix
  keys and were never placed in the same candidate block.
* **Phonetic near-misses** - "Schmidt" and "Schmit" differ at character 7 but
  sound alike; prefix blocking misses them.

## Work Log

- 2026-06-11: Created `src/aryx/resolution/blocking.py`
  - Moved `normalize()` and `_WS` regex here (was in classical.py) to break

  - Implemented `soundex()` inline using the standard A-Z maketrans table

  - Implemented `_keys_for(text)` emitting three keys:
    `prefix:<first4>`, `tokens:<sorted-token-set[:32]>`, `soundex:<code>`.
  - Implemented `MultiKeyBlocker(max_block_size=5000)` with cap: blocks
    exceeding the cap are logged as WARNING and dropped to prevent O(n^2)
    blowup on degenerate data.

  - Removed `normalize`, `_WS`, old `block()` implementation.
  - Added `from aryx.resolution.blocking import MultiKeyBlocker, normalize`.
  - Added shim: `def block(records, max_block_size=5000)` delegates to
    `MultiKeyBlocker` - existing callers (run.py) are unaffected.
- 2026-06-11: Created `tests/test_blocking.py` with 13 tests covering all
  specified assertions plus regression tests for classical helpers.
