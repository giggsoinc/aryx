## Problem

`golden_record()` in `cluster.py` merged by "first non-empty wins" with no weighting, no conflict detection, and no provenance trail. High-confidence pair scores from the scoring stage were computed but discarded — the merge step was blind to them.

## Root Cause (file:line)

- `src/aryx/resolution/cluster.py:golden_record` — flat first-non-empty merge
- `src/aryx/resolution/run.py` — pair scores collected during scoring but never passed to merge
- `src/aryx/models.py:ResolvedEntity` — no `provenance` field

## Fix

1. **`src/aryx/resolution/survivor.py`** (new) — `survivors(payloads, record_ids, pair_scores)`:
   - Empty `pair_scores` → legacy first-non-empty merge (backward compat)
   - With scores: weight = avg pairwise score per contributing record; highest-weight record's value wins per attribute
   - Conflict logged as WARNING when >1 distinct value exists for an attr
   - Returns dict with `_provenance: {attr: record_id}`

2. **`src/aryx/resolution/cluster.py`** — `golden_record()` becomes a shim; `golden_record_weighted()` delegates to `survivors()`

3. **`src/aryx/resolution/run.py`** — collects `pair_scores` during scoring loop; calls `golden_record_weighted`; extracts `_provenance` → `entity.provenance`

4. **`src/aryx/models.py`** — `ResolvedEntity.provenance: dict[str, int] | None = None`

## Acceptance Gate

- `test_provenance_present` — `_provenance` maps each attr to contributing `record_id`
- `test_conflict_logs_warning` — WARNING emitted on value conflict
- `test_legacy_fallback_first_non_empty` — old `golden_record()` shim works unchanged
- 7/7 unit tests pass
