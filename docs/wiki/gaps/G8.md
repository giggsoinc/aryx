## What Was Built

- **Migration 0022** — `updated_at` on `aryx_entity`,
  `aryx_projection_state` (watermark per workspace),
  `aryx_projected_entity` (graph-known ids side table). Dirty-set
  computation stays in Postgres — the graph is never diffed.
- **`store/projection_store.py`** — watermark, dirty entities
  (updated since watermark), tombstones (side table EXCEPT entity table),
  provenance/relationships for the dirty set.
- **`FalkorStore.remove_entity`** — DETACH DELETE for tombstones.
  `add_entity` already MERGEs on `{id}` — upserts were pre-paved by the
  stable-IRI work.
- **`project.py`** — `project_incremental()` (never calls `clear()`; graph
  stays queryable) and `project_auto()`: incremental when a watermark exists
  and dirty < 30% of entities (`ARYX_PROJECT_DIRTY_MAX`), full rebuild
  otherwise — full rebuild remains the correctness anchor.
- **Dirty-set feeders:** G10's `merge_entity_attributes` and G13's
  `update_entity_attributes` both touch `updated_at`.
