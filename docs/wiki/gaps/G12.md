# G12 — Shared psycopg3 Connection Pool

| Field | Value |
|---|---|
| **Status** | DONE |
| **Owner** | pool-fitter |
| **Branch** | aryx-g12-pool |
| **Linked files** | `src/aryx/store/pool.py`, all 10 store files, `src/aryx/api/main.py`, `requirements.txt` |
| **Tests** | `tests/test_pool.py` — 4/4 passing |

## Problem

Every store opened a dedicated `psycopg.connect()` on init and held it for the process lifetime — no pooling, no reuse. Under concurrent API load each store instance leaked a connection; long-running processes exhausted Postgres's `max_connections`.

## Root Cause (file:line)

- All `*Store.__init__` methods: `self._conn = psycopg.connect(dsn)` (per-store, per-request)
- `requirements.txt`: only `psycopg[binary]` — `psycopg_pool` not declared

## Fix

1. **`requirements.txt`** — `psycopg[binary]~=3.2` → `psycopg[binary,pool]~=3.2`

2. **`src/aryx/store/pool.py`** (new) — singleton `ConnectionPool` per DSN:
   ```python
   def get_pool(dsn, min_size=2, max_size=10) -> ConnectionPool  # cached by DSN
   def close_all() -> None  # closes all pools, called at shutdown
   ```

3. **10 stores rewritten** (`ask_history_store`, `rule_store`, `version_store`, `mcp_token_store`, `job_store`, `chunk_store`, `entity_store`, `ontology_store`, `axiom_store`, `postgres_store`):
   - Removed: `self._conn = psycopg.connect(...)`, `self._conn.commit()`, `close()` methods
   - Added: `self._pool = get_pool(dsn)`
   - All methods: `with self._pool.connection() as conn: with conn.cursor() as cur: ...`

4. **`src/aryx/api/main.py`** — `_lifespan` calls `close_all()` on shutdown

5. **`migrate.py` intentionally untouched** — runs once at startup, not under load

## Acceptance Gate

- `grep -rn "psycopg.connect" src/aryx/store/` → only `migrate.py`
- 4/4 pool unit tests pass (singleton, different-DSN, close-all, recreate-after-close)
