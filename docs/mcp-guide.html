MCP process

Drive Aryx from chat.

Aryx ships 21 Model Context Protocol tools that let any MCP host (Claude Desktop, Cursor, Continue, your own agent loop) walk a user through the entire onboarding and querying flow without ever touching the web UI. This is the process — what to call, when, and how.

What MCP gives you

Two surfaces for one knowledge graph.

Same Postgres, same FalkorDB, same brief. Pick the right surface for the audience — or use both side by side.

The Aryx web UI

Brand-faithful Next.js app at :3000. Three surfaces: /start (guided wizard), / (Ask with citations), /model (modelling canvas). Designed for the domain expert who wants pixels.

Best for: live demos, hands-on configuration, visual exploration.

The Aryx MCP server

SSE endpoint at :8765/sse exposing 21 tools. Same wire shape as /admin/* and /ontology/* REST, but as discoverable tools an LLM can plan with.

Best for: agent automation, chat-first interfaces, scripting.

The flow

Five steps. Each one is one or two MCP tool calls.

This is the exact ordering an agent should follow to take a brand-new workspace to "ask anything" — no web UI required.

1
Pick or create a workspace
Isolated tenant per workspace — own LIST partitions, own FalkorDB graph.
workspace_list → all workspaces workspace_create → new isolated tenant workspace_select → confirm + load
2
Capture the brief
Goals + objectives + scope. Agent quizzes the user; AI drafts; agent confirms.
brief_get → read current + next_question brief_draft → AI fills 5 fields from a sentence brief_set → patch one field at a time brief_save → persist + depth meter
3
Add a data source
Quiz fields per kind. Secret encrypted with Fernet before write. Test connection.
datasource_quiz → field spec per kind datasource_add → mask-only response datasource_test → SELECT 1 / list tables datasource_list → what's registered
4
Answer human-in-the-loop questions
Pipeline pauses when it's not sure. Agent surfaces questions to the user, sends answers back, pipeline unblocks.
ingest_status → pipeline progress ingest_questions → pending HITL queue ingest_answer → resolve, unblock entities_preview → what's been resolved
5
Query and export
Ask in plain English, request actions (always pending for human approval), export the ontology to any destination.
ask → grounded NL query act → request entity mutation (HITL) ontology_get → types + relationships ontology_export → Postgres / MySQL / Neo4j / RDF
The 21 tools

Discoverable, typed, documented in-band.

Every tool ships with a structured input schema and a prose description. The agent doesn't need our docs — it can pick the right tool just from the MCP capability list.

Read & act 3

listEvery workspace with entity/relationship counts + type breakdown. Always call first.
askNatural-language question over one workspace. Returns grounded answer + cited entity ids + workspace overview as context.
actRequest an action on an entity. Always-pending — agent mutations never auto-apply.

Workspace 3

workspace_listAll workspaces with id, name, description, brief, timestamps.
workspace_createNew isolated tenant. Owns LIST partitions + named FalkorDB graph.
workspace_selectConfirm a workspace exists; returns its row.

Brief 4

brief_getCurrent brief + readiness depth (Generic NER → Grounded → Sharp → Expert) + next_question for the quiz.
brief_draftAI-draft all 5 fields from a seed sentence and/or document text.
brief_setPatch ONE field. Strings auto-split to lists for objectives/roles.
brief_savePersist atomically. Returns depth + next_question.

Datasource 5

datasource_quizField spec for a kind (postgres, mysql, oracle, files, rest, ...). Drives the agent's prompt.
datasource_addPlaintext secret travels at request scope only; encrypted via Fernet on the server, mask-only in the response.
datasource_listPer-workspace list. Never returns ciphertext.
datasource_testDecrypt + open + ping. Always-pending failure, never silent.
datasource_deleteHard delete row + cipher. Audit-logged.

HITL ingest 4

ingest_statusPipeline stage progress + pending question count.
ingest_questionsOpen clarifying questions ("Customer or Account?") with AI-suggested answers.
ingest_answerResolve one question; pipeline unblocks instantly.
entities_previewSlice of nodes + edges so the agent can confirm visually before full projection.

Ontology 2

ontology_getJSON-LD of approved types + relationships + axioms + rules.
ontology_exportAdapter to Postgres / MySQL / Neo4j / Turtle / JSON-LD. Returns rows written + masked DSN.
Connect

Three paths from chat to Aryx.

Pick by the audience. Claude Desktop is the fastest demo today; Cursor & Continue use the same config shape; claude.ai web requires HTTPS in front.

Recommended
💻

Claude Desktop

~5 minutes · plain HTTP works

Local Mac app. Bridges via mcp-remote over npx — no extra install. Talks plain HTTP to your EC2 MCP endpoint.

📝

Cursor & Continue

~5 minutes · same config shape

IDEs with MCP support read the same mcpServers JSON shape. Aryx tools appear in the inline assistant.

🌐

claude.ai web

~15 min · needs HTTPS

Web Claude blocks raw-HTTP custom MCPs. Front the server with Cloudflare tunnel for a free HTTPS URL, paste it into Settings → Connectors.

Path A: Claude Desktop (the recommended demo)

# 0. Open EC2 SG inbound for 8765/tcp (same as you did for 3000 and 8088).

# 1. Verify reachability from your Mac:
curl -m 6 -i http://ec2-3-91-73-197.compute-1.amazonaws.com:8765/sse
# Expect Content-Type: text/event-stream.

# 2. Edit ~/Library/Application Support/Claude/claude_desktop_config.json:

{
  "mcpServers": {
    "aryx": {
      "command": "npx",
      "args": [
        "-y", "mcp-remote",
        "http://ec2-3-91-73-197.compute-1.amazonaws.com:8765/sse"
      ]
    }
  }
}

# 3. Quit Claude Desktop fully (⌘Q), reopen. Click the 🔌 icon.
# Should show 21 tools under "aryx".

Path C: claude.ai web via Cloudflare tunnel

# On the EC2 instance:
sudo curl -L -o /usr/local/bin/cloudflared \
  https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64
sudo chmod +x /usr/local/bin/cloudflared
cloudflared tunnel --url http://localhost:8765

# Prints a URL like https://abc-def.trycloudflare.com
# Use that URL + /sse in claude.ai → Settings → Connectors → Custom MCP.

Security posture

What the MCP server will and won't do.

Guarantees

  • Datasource secrets are Fernet-encrypted server-side before any write. Plaintext lives only in the request scope.
  • Responses include masks only — the last 4 chars of a SHA-256 hash of the cipher.
  • Every decrypt is audit-logged with actor, timestamp, and reason.
  • Agent-initiated act requests always pend for human approval — regardless of an action's auto-apply flag.
  • HITL questions go through the same queue as the web UI — no out-of-band side channel.
  • The MCP server is a thin shell over the same FastAPI endpoints — no parallel logic to drift.
