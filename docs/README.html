How it works

Four steps. You stay in control.

No data model upfront. State your goals; Aryx proposes the structure, you approve and tweak.

1

Tell Aryx what you want to figure out

List 2–5 questions or goals. Plain English. The nouns become your data model.

2

Point Aryx at your data

Database, files, APIs — whatever you have. Mix and match.

3

Aryx proposes the model

Records get deduped, things get linked. You answer anything ambiguous from a side drawer.

4

You ask, Aryx answers

Plain English questions. Every answer cites the records behind it.

Quickstart

From clone to running in under five minutes.

Docker Compose brings up Postgres, FalkorDB, Ollama, FastAPI, MCP, Streamlit, and the Next.js app.

# 1. Clone + bring up the stack
git clone https://github.com/giggsoinc/aryx.git
cd aryx
docker compose up -d

# 2. Wait for healthy (≈ 60s on a warm pull)
docker compose ps

# 3. Open the wizard — Aryx will guide you the rest of the way
open http://localhost:3000/start

# Optional — connect Claude Desktop via MCP (21 tools)
# Add to ~/Library/Application Support/Claude/claude_desktop_config.json:
  "aryx": { "command": "mcp-proxy", "args": ["http://localhost:8765/mcp"] }
